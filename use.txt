# NTC - Navigate, Tree, Cat
## Version 2.2.0 - Complete User Guide

NTC is a command-line tool that combines directory navigation, tree viewing, file concatenation, backup/restore, fuzzy search, built-in text editor, syntax highlighting, math expression evaluator, and a dinosaur runner game. It works on Windows, Linux/WSL, macOS, and Termux (Android).

## INSTALLATION

### Windows
- Download `ntc.exe` from GitHub Releases
- Run the Inno Setup installer
- Or manually place `ntc.exe` in a folder in your PATH
- Or install via winget: `winget install NuengCoder.ntc`

### Linux (Ubuntu/Debian)
- Download `ntc_2.2.0-1_amd64.deb`
- Run: `sudo dpkg -i ntc_2.2.0-1_amd64.deb`

### Linux (Other distributions)
- Download `ntc-v2.2.0-linux-x86_64.tar.gz`
- Extract: `tar -xzvf ntc-v2.2.0-linux-x86_64.tar.gz`
- Copy to PATH: `sudo mv ntc /usr/local/bin/`

### macOS (Intel & Apple Silicon)
- Download `ntc-v2.2.0-macos-universal.tar.gz`
- Extract: `tar -xzvf ntc-v2.2.0-macos-universal.tar.gz`
- Copy to PATH: `sudo mv ntc /usr/local/bin/`

### Termux (Android)
- Download `ntc-v2.2.0-aarch64-linux-android.tar.gz`
- Extract: `tar -xzvf ntc-v2.2.0-aarch64-linux-android.tar.gz`
- Copy: `cp ntc $PREFIX/bin/`
- `chmod +x $PREFIX/bin/ntc`

### npm (cross-platform)
- `npm install -g @nuengcoder/ntc`

## TASK RUNNER (ran)

ntc includes a built-in Make-like task runner using `NTCRANFILE.toml`:

### Creating a ranfile:
  ran init                   - Create NTCRANFILE.toml template
  ran init --all             - Create with sample targets

### Running targets:
  ran <target>               - Execute target with dependency resolution
  ran build                  # Run build target (auto-runs dependencies first)
  ran test                   # Run test target

### Managing targets:
  ran list                   - List all targets with dependency chains
  ran ls                     - Same as list
  ran help                   - Show ranfile syntax and examples
  ran deinit                 - Remove NTCRANFILE.toml

### NTCRANFILE.toml format:
  [vars]
  CC = "gcc"
  FLAGS = "-Wall -O2"

  [targets.build]
  deps = ["clean"]
  cmd = "{{CC}} {{FLAGS}} -o program main.c"

  [targets.test]
  deps = ["build"]
  cmd = "./program --test"

  [targets.clean]
  cmd = "rm -f *.o program"

### Features:
  - Topological sort: dependencies run before their dependents
  - Cycle detection: circular dependencies are rejected
  - Variable expansion: `$(VAR_NAME)` or `{{VAR_NAME}}` syntax
  - Commands go through ntc's alias expansion system
  - Integrated with top-level `init`/`deinit` commands

### Top-level project initialization:
  init                       - Create both NTCRANFILE.toml + ntconfig.toml
  init --ran                 - Create only NTCRANFILE.toml
  init --local               - Create only ntconfig.toml
  init --all                 - Create both with sample content
  deinit                     - Remove project files (asks per file)
  deinit --ran               - Remove only NTCRANFILE.toml
  deinit --local             - Remove only ntconfig.toml

## FILE & DIRECTORY SEARCH

### Search files (exact -> partial -> fuzzy):
  fs <pattern>               - Search files using configured depth
  fs <pattern> -d <n>        - Search files with custom depth
  fs main.c                  # Exact match
  fs mian.c                  # Fuzzy match: suggests main.c (96% match)
  fs test                    # Partial match: finds test_helper.c

### Search directories:
  ds <pattern>               - Search directories using configured depth
  ds <pattern> -d <n>        - Search directories with custom depth
  ds src                     # Exact match
  ds srcd                    # Fuzzy match: suggests src (94% match)
  ds test                    # Partial match: finds test_utils/

### Combined search (files + directories):
  locate <pattern>           - Search both files and directories
  locate <pattern> -d <n>    - Combined search with custom depth

### Search + navigate (fgo):
  fgo <pattern>              - Search files, select one, navigate to its parent directory
  fgo <pattern> -d <n>       - Search with custom depth

### Search + display (fsc):
  fsc <pattern>              - Search files, select one, display its contents
  fsc <pattern> -d <n>       - Search with custom depth

### Wildcard / Glob Search:
  fs, ds, fsc, fgo, and locate also support wildcard patterns:
  - `*` or `%` — matches any sequence of characters (including empty)
  - `_` — matches any single character
  - `$` — anchors the pattern to the end of the name
  - Examples:
    fs *.rs                        # All Rust files
    fs main_*                      # Files starting with "main_"
    ds src$                        # Directories ending with "src"
  - When wildcards are detected, exact/partial/fuzzy matching is bypassed

### Grep content search (gs):
  gs <pattern>               - Search file contents (case-insensitive substring match)
  gs <pattern> -d <n>        - Search with custom depth
  gs "TODO"                  # Find all files containing "TODO"
  gs "fn main"               # Find Rust main functions
  Results show: file path, line number, and matching line content

### How search works:
  1. Wildcard patterns: `*`, `%`, `_`, `$` — glob-style matching
  2. Exact matches (full filename equality, case-insensitive)
  3. Partial matches (filename contains the pattern)
  4. Fuzzy matches (Jaro-Winkler similarity >=75% for files, >=72% for directories)
  5. Shows top 3 fuzzy suggestions when no exact or partial match found

## BUILT-IN TEXT EDITOR (ntcEditor)

ntc includes a full-featured built-in text editor for editing files directly from the shell.

### Opening the editor:
  ne                           - Open scratch buffer in current directory
  ne <file>                    - Open file for editing
  ne <path>/<file>             - Open file at specific path
  ne .<file>                   - Open hidden file (e.g., ne .gitignore)
  ne <dir>                     - Open scratch buffer inside directory
  ne --init <file>             - Create file from template (supports .rs, .py, .c, etc.)
  ntceditor <file>             - Same as ne

### Editor features:
  - **Multi-tab editing** with tab bar, tab switching (Ctrl+PgUp/PgDn), and buffer stack (Ctrl+Tab)
  - **Tab close buttons** (✕) — click any tab's ✕ to close it; closing the last tab creates a scratch buffer
  - **Sidebar** file explorer (Ctrl+B) with expand/collapse folders and directory icons (▶/▼)
  - **Sidebar vertical scroll bar** — `█`/`░` thumb/track with click-to-jump and mouse wheel support
  - **Multi-cursor editing** — Ctrl+D to add cursor at next occurrence, Ctrl+G to jump through matches
  - **Undo/redo** with unlimited history
  - **Mouse support** (click to position cursor, scroll, drag scroll bars)
  - **Line numbers** and gutter with configurable width
  - **Syntax highlighting** for 45+ languages including Inno Setup (.iss) and lock files (.lock)
  - **Clipboard operations** (cut, copy, paste) with multi-line support
  - **Word wrap** and horizontal scrolling
  - **Auto-indentation** and bracket pairing
  - **Search within file** (Ctrl+F) with next/previous navigation
  - **`:gs <pattern>`** in command mode — grep file contents across project
  - **`:run <command>`** — execute shell commands with live streaming output in split panel
  - **`:gosc`** in command mode — continuous subdirectory navigation within editor
  - **File finder** (Ctrl+P / Ctrl+T) — fuzzy search files across project and open in new tab
  - **Command mode** (`:q`, `:w`, `:wq`, `:q!`) for quit/save operations
  - **Crash recovery** — periodic SHA-256 hashed snapshots with auto-recovery prompt
  - **Persistent sessions** — cursor, scroll, buffer stack restored on re-entry
  - **Auto-completion** for .ntc.math files (functions, constants, keywords)
  - **Configurable** tab width, color theme, auto-save, syntax toggle (editor.toml)
  - **Template creation** — `ne --init file.rs` creates file from template by extension

## RUN ALIASES (Custom Commands)

### Create aliases:
  ral add <name> "<command>"       - Create new alias
  ral add <name>(x) "<cmd $x>"     - Create parameterised alias (single arg)
  ral add <name>(x,y) "<cmd $x $y>" - Create parameterised alias (multiple args)

  Example: ral add ll "ls -la"
  Example: ral add py "python test.py"
  Example: ral add runc(x,y) "gcc -o $y $x.c && ./$y"

### Use aliases:
  <alias>                          - Just type the alias name
  <alias>(<arg>)                   - Run parameterised alias
  run <alias>                      - Run with 'run' prefix
  r <alias>                        - Shortcut for run

### Manage aliases:
  ral edit <name> "<new_command>"  - Update existing alias
  ral rnm <old> to <new>           - Rename an alias
  ral rm <name>                    - Remove alias
  ral list                         - Show all aliases
  ral cls                          - Clear ALL aliases (asks confirmation)
  ral cls --force                  - Clear ALL aliases (no confirmation)

### Export/Import aliases:
  ral export --all <name>          - Export all aliases to <name>.ntc.ral
  ral export --select <name>       - Select which aliases to export
  ral import <file.ntc.ral>        - Import aliases from .ntc.ral file

### Alias Rules:
  - Cannot start with @ or #
  - Cannot be reserved commands (go, view, txt, bkup, pldw, unpd, fs, ds, etc.)
  - Use letters, numbers, hyphens, underscores
  - Supports && (use quotes: ral add fb "dal && frr")

## LOCAL PROJECT CONFIGURATION

Place an `ntconfig.toml` file in any directory to override settings for that project:

### What can be overridden:
  - Ignored directories, extensions, and files
  - Extra supported files and extensions
  - Run aliases (project-specific commands)

### What stays global:
  - Teleport savepoints
  - Output path, max depth, line numbers, color setting
  - History settings, file watcher
  - Thread count

### How it works:
  1. Create ntconfig.toml: `gencg` or `gencg --all`
  2. Edit the file with your project settings (or `ne ntconfig.toml`)
  3. Navigate into the directory - settings auto-load
  4. Leave the directory - global settings restore automatically

### Example:
  # In project directory
  ntc [project]> ignore build
  ✓ Now ignoring directory: build (local config)

  ntc [project]> ral add test "cargo test"
  ✓ Alias 'test' -> 'cargo test' (local config)

  ntc [project]> cd ..
  # Global settings restored

## MATH EXPRESSION EVALUATOR

ntc includes a full math expression evaluator with built-in functions and constants.

### Interactive mode:
  math <expression>                - Evaluate a math expression
  math 3+4*5                      # → 23
  math sin(PI/2)                  # → 1
  math sqrt(144)                   # → 12
  math print("hello world")       # → hello world
  math rand(1,100)                 # → 42

### Named arguments and string literals:
  math print("the answer is", 42)  # → the answer is 42
  math print("line1\nline2")       # multi-line with escape sequences

### Timer:
  math timer                       - Start lap timer
  math timer 10                    - Countdown 10 seconds

### File mode (.ntc.math):
  math <file>.ntc.math             - Compile and run a math script file
  # Example file content:
  #   x = 42
  #   print("x is", x)
  #   return x * 2

### Built-in functions:
  print:   print values (like Python — accepts strings and numbers)
  trig:    sin, cos, tan, cot, sec, csc
  inverse: arcsin/asin, arccos/acos, arctan/atan, arccot/acot, arcsec/asec, arccsc/acsc
  math:    sqrt, pow(x,y), abs, floor, ceil, ceiling, round, ln/log, log2, log10
  agg:     sum, min, max, avg/average/mean
  conv:    toBinary, toHex, to8/toOctal, toDecimal, toHB/toHumanBytes
  rand:    rand(min, max)

### Constants:
  PI, E/EXP, PHI, TAU

### User-defined functions:
  math fun add <name>(<params>) = <body>     - Define a function
  math fun edit <name> = <body>              - Update a function
  math fun rm <name>                         - Remove a function
  math fun info <name>                       - Show function details
  math fun ls                                - List all functions

  Example:
    math fun add square(x) = x^2
    math square(5)                           # → 25

### Command-line flag:
  ntc --math "3+4*5"                         - Evaluate from command line

## TIPS & BEST PRACTICES

1. Use `bkup` before making major changes to your project
2. Use `pldw` to restore and `unpd` to revert if you made a mistake
3. Use `diff 1` to see what changed since your last backup
4. Use `fs` with fuzzy matching for quick file finding (even with typos!)
5. Use `fgo` to search for a file and jump straight to its directory
6. Use `fsc` to search for a file and view its contents immediately
7. Use `locate` for combined file + directory search
8. Use `ne` (ntcEditor) for quick file edits without leaving the shell
9. Use `ne --init <file>` to create new files from templates
10. Use `go to <tp_name>` for instant navigation to frequently used directories
11. Use `view -s -d <n>` to quickly see folder sizes at specific depth
12. Use teleport points for frequently accessed directories
13. Create run aliases for long or complex commands
14. Set line numbers ON when reviewing code files
15. Use --cp to quickly share directory structures
16. Ignore build directories (target, node_modules) for cleaner trees
17. Use gosc for rapid directory traversal
18. Chain commands with && for automation
19. Export reports in JSON for integration with other tools
20. Use PDF/DOCX/XLSX for professional report distribution
21. Use ntconfig.toml for project-specific settings
22. Back up your config with resetcg before major changes
23. Use `theme export` to share custom themes across machines
24. Use `gs` to quickly find where a function or variable is used across the project

## TROUBLESHOOTING

### Command not found:
  - Ensure ntc is in your PATH
  - On Linux: Check ~/.config/ntc/config.toml exists
  - On Windows: Check %APPDATA%\ntc\config.toml

### Clipboard copy not working (Linux):
  - Install xclip: sudo apt install xclip
  - Or install wl-clipboard: sudo apt install wl-clipboard

### File watcher not working:
  - Run: ntc --watch ON
  - Restart ntc to activate

### Teleport points lost:
  - Check config file exists and is readable
  - Use 'tp list' to verify savepoints

### Local config not loading:
  - Verify ntconfig.toml exists in current directory
  - Check file has valid TOML syntax (no errors in VSCode)
  - Run 'showcg' to see active config source

### Backup failed (file too large):
  - ntc skips files >50 MB by default
  - See ignore.txt in the backup directory for details

### Undo not working:
  - `unpd` only works after a `pldw` restore
  - Undo history is cleared after successful undo

### Editor not opening:
  - Ensure the file path is valid and writable
  - Use `ne` without arguments to test editor startup
  - ntcEditor will fall back automatically if no external editor is found

### Report generation failed (PDF/DOCX/XLSX):
  - Ensure output directory exists and is writable
  - Check available disk space
  - These formats may require additional memory for large directories

## PLATFORM DIFFERENCES

### Windows-only:
  - godrive command (drive letter navigation)
  - Inno Setup installer
  - Windows-style paths (C:\Users)

### Linux/WSL-only:
  - .deb package installation
  - Config at ~/.config/ntc/
  - Unix-style paths (/home/user)

### macOS:
  - Universal binary (Intel + Apple Silicon)
  - Config at ~/.config/ntc/
  - Unix-style paths (/Users/username)

### Termux (Android):
  - pkg install support
  - Config at $PREFIX/var/lib/ntc/
  - Unix-style paths (/data/data/com.termux/files/home)

### Cross-platform (works everywhere):
  - All navigation commands
  - Backup & restore (bkup, pldw, unpd, diff)
  - File & directory search (fs, ds, locate, fgo, fsc)
  - All report formats (TXT, JSON, MD, HTML, PDF, DOCX, XLSX)
  - Teleport savepoints (including go to / cd to)
  - Run aliases (with && support, multiple args)
  - File viewing
  - Ignore/care system
  - Local project configs
  - Built-in text editor (ntcEditor)
  - Syntax highlighting
